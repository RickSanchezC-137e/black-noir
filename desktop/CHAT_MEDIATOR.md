# Чат «Модуль взаимодействия» — три канала с постоянной памятью

Вкладка **ЧАТ** содержит переключатель между тремя каналами связи. Каждый канал
ведёт собственную **постоянную историю** (хранится по `session_id`; на клиенте —
в `localStorage`, на сервере — в таблицах `sessions`/`messages` + долгосрочная
память ChromaDB). Голосовая строка, эквалайзер, окно-«лицо» и вложения общие для
всех каналов.

## 1. ПЕРЕДАТЧИК (посредник владелец ⇄ ядро) — канал по умолчанию

Цель: чтобы владелец и ядро **лучше понимали друг друга** и ядро **не нагружалось
мелкими разрозненными запросами**.

- **owner → ядро.** Реплики владельца можно копить в **буфер** (Enter — добавить
  строку в буфер, кнопка ⮞ — «собрать и отправить»). Передатчик берёт пачку
  (возможно отрывочных) сообщений по одной теме и формирует **один чёткий,
  полный запрос** к ядру (цель + контекст из истории + критерий результата). В
  чате показывается строкой `→ ядру: …`, что именно ушло в ядро.
- **ядро → owner.** Ядро отвечает развёрнуто; Передатчик доносит владельцу
  **только суть** — кратко, простыми словами (2–5 предложений). Полный ответ
  ядра сохраняется (`core_reply`) и доступен.
- Реализация: `app/core/mediator.py` (`relay()` = форма-задачи → ядро → суть),
  через `claude.chat_as(system, …)`. Маршрут: `POST /api/chat {agent:"mediator"}`.

## 2. ЯДРО — прямой диалог

Прямой чат с оркестратором ядра (`claude.chat` с историей сессии). Без
посредника, для случаев, когда нужно говорить с ядром напрямую.
Маршрут: `POST /api/chat {agent:"core"}`.

## 3. CLAUDE CODE — чат с кодовым агентом

Диалог с headless Claude Code (`claude -p --output-format json`) в каталоге репо,
**read-only** (правки в headless-режиме без одобрения отклоняются — мутации это
работа изолированного Builder’а, не чата). Память между ходами — через session id
CLI (`--resume`). Реализация: `app/core/claude_code.py`. Маршрут:
`POST /api/chat {agent:"claude_code", cc_session}`.

## Контракт `/api/chat`

Вход: `{message, agent, session_id?, cc_session?, target?}`.
Выход: `{reply, session_id, agent, task?, core_reply?, cc_session?}`
— `task`/`core_reply` заполняет Передатчик; `cc_session` — канал Claude Code.

## Память

Постоянство = стабильный `session_id` на канал (клиент хранит его и историю в
`localStorage`), серверная история берётся из `messages` по `session_id` и
передаётся модели как контекст; пользовательские реплики дополнительно идут в
долгосрочную память (`memory.remember`, source `chat:<agent>`).
